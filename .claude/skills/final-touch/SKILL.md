# 11단계 · 완성 (Final Touch)

## 역할
제목(Title)을 최종 확정하고, 독자에게 남길 마지막 여운(Ending)을 점검한다. 글이 하나의 완결된 작품으로서 **물성**을 갖추는지 확인한다.

## 행동 절차

### Step 1. 제목 후보 5종 (유형별 1개씩)

| 유형 | 특징 | 예시 |
|---|---|---|
| 감성형 | 정서적 공명 | "개발자의 글쓰기, 30년의 고백" |
| 직관형 | 핵심 직진 | "코드처럼 글을 쓰는 법" |
| 질문형 | 호기심 자극 | "왜 개발자는 글을 써야 하는가?" |
| 숫자형 | 구체성·실용 | "개발자를 위한 글쓰기 12단계" |
| 은유형 | 시적 함축 | "글쓰기, 리팩토링의 예술" |

각 후보의 강점과 어울리는 플랫폼을 1줄로 메모.

### Step 2. 최종 점검 체크리스트

```
✅ 도입부가 3문장 내에 독자를 끌어당기는가?
✅ 결말이 여운을 남기는가? (명확한 결론 / 열린 질문 중 의도 일치)
✅ 논리 비약 없이 흐름이 자연스러운가?
✅ 작가의 고유한 목소리가 들리는가?
```

각 항목 통과 여부를 ✅ / ⚠️ / ❌로 표시하고, ⚠️·❌는 구체적 위치·수정 제안 첨부.

### Step 3. 기술적 검증

**맞춤법 스캔 (한국어 맞춤법 검사기 기준)**
- 자주 틀리는 어휘 자동 탐지: 되/돼, 안/않, 든/던, 데/대, 율/률 등
- 띄어쓰기 의심 위치 표시
- 외래어 표기 (영어식/일본식 표기 검출)

**표절 검사 알림 (자동 검사 X, 권고만)**
> "⚠️ 인용구나 외부 문헌을 사용한 부분의 출처를 명시했는지 다시 확인해 주세요."

### Step 4. 발행 준비 메시지
> "🎉 작가님, 이 글은 이제 세상에 나갈 준비가 되었습니다.
> 12단계(발행)로 넘어갈까요? 어떤 플랫폼에 올리실지 정하면, 그에 맞춘 패키지를 만들어 드리겠습니다."

## 입력 / 출력

**입력:**
- 10단계 퇴고 완료본

**출력:**
- **제목 후보 5개** (유형별 + 강점 메모)
- **최종 점검 체크리스트 결과** (4개 항목 ✅/⚠️/❌)
- **맞춤법 오류 목록** (있다면)
- 다음 단계(`publishing-meta`) 진행 여부 확인
