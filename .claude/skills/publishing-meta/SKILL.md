# 12단계 · 발행 (Publishing & Meta)

## 역할
글을 독자에게 전달하는 포장 단계. 플랫폼(블로그·브런치·미디엄·출판 원고 등) 성격에 맞는 태그·서문·썸네일 아이디어를 정리한다.

## 행동 절차

### Step 1. 플랫폼 확인
AskUserQuestion으로:
> "어느 플랫폼에 발행하시나요? (브런치 / 미디엄 / 개인 블로그 / 출판 원고 / 기타) — 복수 선택도 가능합니다."

### Step 2. 플랫폼별 차별화 전략

| 플랫폼 | 특화 산출물 |
|---|---|
| **브런치** | 감성적 썸네일 키워드, 첫 문장 미리보기 최적화, 매거진 카테고리 추천 |
| **미디엄** | 영문 태그 5개, 글로벌 독자용 문화 맥락 각주, Member-only/Free 권장 |
| **개인 블로그** | SEO 키워드 5개, 메타 디스크립션 (150자), 슬러그 추천 |
| **출판 원고** | 편집자용 커버레터, 목차 위치 제안, 챕터 분량 균형 점검 |
| **뉴스레터** | 제목 줄(40자), 프리헤더(80자), 클릭 유도 CTA 1줄 |

선택된 플랫폼별로 위 항목을 만들어 제공한다.

### Step 3. 공통 산출물

#### ① SEO 키워드 5개
주제·독자가 검색할 만한 키워드 조합.

#### ② 해시태그 10개
플랫폼 톤에 맞춰 한국어/영문 혼합.

#### ③ 작가의 말 (100자 이내)
독자에게 보내는 짧은 메시지. 1인칭, 친밀한 거리.

#### ④ 뉴스레터용 요약문 (300자)
구조: 후크 1문장 → 핵심 통찰 1문장 → 본문 미리보기 1~2문장 → 클릭 유도 CTA.

#### ⑤ 소셜 미디어 발췌문 3종

| 채널 | 글자 수 | 톤 |
|---|---|---|
| **트위터/X** | 140자 | 핵심 한 줄, 호기심 후크 |
| **링크드인** | 300자 | 전문성 강조, 인사이트 압축, 독자 페르소나 명시 |
| **인스타그램 캡션** | 200자 + 이모지 | 감성적, 줄바꿈으로 시각 호흡 |

#### ⑥ 썸네일 아이디어 (또는 이미지 키워드)
- 디자이너용 brief 1단락
- 색감·구도·핵심 오브제 키워드 5개
- (이미지 생성 AI에 그대로 넣을 수 있는) 영문 프롬프트 예시 1개

### Step 4. 최종 패키지 산출 (예시)

```markdown
# 📦 [글 제목] — 발행 패키지

## 플랫폼: 브런치 + 개인 블로그
- 매거진: 개발 / 에세이
- 슬러그: dev-writing-12-steps

## SEO 키워드
1. 개발자 글쓰기  2. 에세이 쓰는 법  3. 리팩토링 ...

## 해시태그
#개발자글쓰기 #에세이 #리팩토링 #DevLife #WritingForDevs ...

## 작가의 말 (100자)
"30년 코드를 쓰며 깨달은 것: 좋은 글도 좋은 코드처럼 리팩토링이 필요하다."

## 뉴스레터 요약 (300자)
...

## SNS 발췌문
- X: ...
- LinkedIn: ...
- Instagram: ...

## 썸네일 brief
...
```

### Step 5. 마무리
> "🎉 작가님, 발행을 축하합니다! 이 글이 독자들에게 울림을 주기를 바랍니다.
> 다음 글이 떠오르면 다시 1단계(`cartesian-doubt`)에서 시작해 주세요."

## 입력 / 출력

**입력:**
- 11단계 완성본 + 발행 플랫폼 정보

**출력:**
- **플랫폼별 발행 패키지** (위 6개 항목 모두 포함)
- **최종 축하 메시지**
