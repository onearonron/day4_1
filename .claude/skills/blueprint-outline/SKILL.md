# 5단계 · 목차 짜기 (Blueprint Engineering)

## 역할
독자의 호흡을 고려해 글의 설계도를 그린다. 각 꼭지(Chapter)가 유기적으로 연결되도록 구조를 배치한다.

## 행동 절차

### Step 1. 구조 선택 — 작가에게 묻기
AskUserQuestion 으로 선호 구조를 확인. 3개 후보 중 택 1, 또는 "추천해줘".

| 구조 | 특징 | 적합한 글 |
|---|---|---|
| **기승전결** | 동양적, 여운 중시 | 에세이, 회고록 |
| **독자 여정 맵** | Hook → Problem → Insight → Action | 실용 칼럼, 가이드 |
| **3막 구조** | Setup → Confrontation → Resolution | 스토리텔링, 회고 |

### Step 2. 3가지 목차 후보 작성
**모든 구조로 한 번씩** 목차를 짜서 비교 제시. 작가가 직접 고르도록.

각 후보는 다음을 포함:
- 꼭지별 소제목 (본문 함축 + 클릭 유도)
- 각 꼭지별 예상 분량(단어 수 또는 글자 수)
- 한 줄 요약

예시 (독자 여정 맵):
```
[1] 🎣 Hook — "10년 차 개발자의 가장 무서운 적은 누구일까" (300자)
[2] ❓ Problem — 익숙함이 만든 함정: 레거시 의존 (700자)
[3] 💡 Insight — 글쓰기는 코드 리팩토링의 거울 (1,200자)
[4] 🎯 Action — 오늘 시작할 30분 글쓰기 루틴 (500자)
```

### Step 3. 소제목 검증
각 소제목에 대해:
- 본문 내용을 함축하는가?
- 클릭/스크롤 유도 매력이 있는가?
- 키워드와 호기심이 균형을 이루는가?

### Step 4. 분량 권장값
- 도입부: 전체의 10~15%
- 본론: 60~70%
- 결말: 15~20%

### Step 5. 추천
3개 후보 중 **가장 추천하는 1개**를 선택 이유와 함께 제시. 작가가 동의하면 확정.

## 입력 / 출력

**입력:**
- 4단계의 우선순위별 자료 배치도

**출력:**
- **3가지 목차 후보** (각기 다른 구조 적용)
- **각 꼭지별 예상 분량**
- **추천 목차 1개** (선택 이유 포함)
- 다음 단계(`voice-persona`) 진행 여부 확인
